Introduction

This API was made for a sport event called BJS. With this API all events, results and classes can be managed.

Usage of HTTP Verbs

HTTP Verb Usage

GET

GET is used to retrieve information/get objects from the api

POST

POST is used to create new objects

DELETE

DELETE is used to delete objects at the api

PUT

PUT is used to replace information regarding an existing object

PATCH

PATCH is used to change and update information of an existing object

User management

This API puts value on security (authentication and authorization). That means it protects the students' sport results, so only those with given access can see and change the information.

Login

To authenticate as an user, this API uses Basic Authentication meaning the user information has to be passed in the HTTP-Header.

Registration

User accounts can only be created by admins. See * for more information.

User Rights

In general only authenticated users can communicate with this API. To get and change information this user must have the privileges to do so. Also only admins can create and delete objects.

Classes

Authentication and Authorization

Only admins can create and delete classes. To get privileges for reading and changing information look at user privileges.

Class as an object

Create new class

Only className is required for creating new classes. But the class name has to be unique. All other fields can optionally be defined.

Parameter Type Optional Description

className

String

false

The class' name

classTeacherName

Host: localhost:8080

Students

Authentication and Authorization

To get privileges for reading, changing, deleting and creating students look at user privileges. Who has rights to access the students is determined by which class he/she belongs to. Admins have the right for all students.

Student as an object

Students have to belong to a class. This is essential for many operations like who has access to manage students which student.

Create student

It is only possible to create a student, belonging to a class the user has rights to. In order to create, send a POST request to /api/v1/students with the information describing the user.

Request fields

Parameter Type Optional Description

firstName

String

false

The student’s first name

lastName

String

false

The students’s last name

birthDay

String

false

The student’s birth day

female

Boolean

false

Whether or not the student is female

schoolClass

String

false

URL to school class the student belongs to

Example request

POST /api/v1/students HTTP/1.1
Accept: application/json
Host: localhost:8080
Content-Length: 138

{
  "firstName": "Simon",
  "lastName": "Schwarz",
  "female": false,
  "birthDay": "2002-01-27",
  "schoolClass": "/api/v1/classes/547"
}
